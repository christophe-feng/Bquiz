# 健康促進網 程式碼簡化與優化指南

依照 173002B10.pdf 第 37 頁到第 53 頁的內容要求，針對現有的 PHP 程式碼進行了精簡與直覺化的分析，以下是整理出的簡化建議：

## 1. 資料庫操作 (api/db.php)

### 簡化資料庫存取
原本的 `all`, `find`, `count` 等函式中有許多重複的 SQL 組合邏輯。
- **優化點**：將 `WHERE` 條件的組合提取成一個私有方法，減少重複程式碼。
- **簡化寫法**：使用 `array_keys()` 與 `join()` 快速組合 `INSERT` 與 `UPDATE` 語句，避免手動拼接字串。

### 訪客統計邏輯
- **原寫法**：判斷 `$_SESSION['total']` 是否存在，再透過 `find` 與 `save` 更新。
- **優化建議**：可以使用 `INSERT INTO ... ON DUPLICATE KEY UPDATE` 的 SQL 語法，在資料庫層級一次完成「不存在則新增，存在則更新」。

## 2. 主架構與顯示 (index.php / back.php)

### 日期與時間顯示
- **原寫法**：`<?= date(" m 月 d 日 l"); ?>`
- **優化建議**：`date("n 月 j 日 l")` 可以讓月份與日期不補零，視覺上更精簡直覺。

### 登入狀態判斷
- **原寫法**：巢狀的 `if` 判斷 `SESSION['login']` 與是否為 `admin`。
- **優化建議**：利用 PHP 的三元運算子 `($condition ? true : false)` 或 `??` 運算子，能將原本 5-10 行的判斷縮減為 2-3 行。

### 頁面包含邏輯
- **優化建議**：`$do = $_GET['do'] ?? 'main';` 搭配 `file_exists()` 檢查後直接 `include`，不需使用長串的 `switch` 或多重 `if`。

## 3. 功能模組 (front/news.php / api/good.php)

### 分頁功能
- **優化建議**：分頁邏輯（計算總頁數、起始位置、產生活動連結）在多個頁面都會用到，應封裝成一個通用函式，傳入資料表名稱與每頁數量即可。

### 點讚 (Good) 邏輯
- **原寫法**：先 `find` 文章，修改 `good` 欄位後再 `save`。
- **優化建議**：直接執行 SQL `UPDATE post SET good = good + 1 WHERE id = ...`。這樣可以減少一次資料庫查詢，且避免並行操作時的數據不一致。

## 4. JavaScript 與 AJAX

### 登入檢查
- **原寫法**：先呼叫 `chk_acc.php` 再呼叫 `chk_pw.php`。
- **優化建議**：應統一由一個 `login.php` API 處理，直接回傳登入成功與否的狀態碼（如 1:成功, 0:密碼錯, -1:帳號錯），減少網路連線次數。
